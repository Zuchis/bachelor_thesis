Devido à diferença de desempenho entre os processadores e o acesso à 
memória principal, muitas aplicações modernas têm 
sofrido com um problema conhecido como gargalo do processador-memória, no 
qual o processador fica ocioso aguardando dados serem buscados na memória. 
Aplicações gráficas, principalmente jogos eletrônicos, são notórias por 
sofrerem deste problema pela elevada quantidade de dados necessários para o
seu funcionamento. Este trabalho propõe a implementação de um motor de jogos 
(\textit{game engine}) utilizando um conjunto de práticas de programação, cuja 
premissa é escrever um código que otimize a leitura da memória através da 
correta estruturação de dados. Este conjunto de práticas é conhecido como projeto 
orientado a dados, do inglês \textit{Data Oriented Design} (DOD), e possui potencial 
para amenizar esse problema do gargalo de desempenho. Para testar a eficiência do DOD, 
foram desenvolvidas duas aplicações diferentes, uma utilizando DOD, e outra com 
programação orientada a objetos, considerada padrão no mercado, e seus desempenho foram 
comparados. Os resultados mostram que o DOD é uma alternativa mais robusta e apresenta 
um desempenho mais favorável na maioria dos casos. Por fim foram feitas as conclusões 
a respeito do DOD e sua utilidade.
