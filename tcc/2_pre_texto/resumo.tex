Devido à diferença de desempenho entre os microprocessadores e a velocidade de acesso 
à memória principal, muitas aplicações de computador modernas têm sofrido com um problema 
conhecido como gargalo do processador-memória, no qual o processador fica ocioso aguardando 
dados serem buscados na memória. Aplicações gráficas, principalmente jogos eletrônicos, são 
notórias por sofrerem deste problema pela demasiada quantidade de elementos diferentes que as 
constituem. Este trabalho propõe a implementação de um motor de jogos (uma \textit{engine}) 
utilizando um conceito de estruturação de código, a modelagem orientada a dados, com a 
finalidade de amenizar este problema de gargalo causado pelo acesso constante à memória.
O trabalho será implementado na linguagem de programação Rust, linguagem a qual visa 
velocidade e uso eficiente e seguro da memória.
