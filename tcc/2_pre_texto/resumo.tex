Devido à diferença de desempenho entre os processadores e o acesso à 
memória principal, muitas aplicações modernas têm 
sofrido com um problema conhecido como gargalo do processador-memória, no 
qual o processador fica ocioso aguardando dados serem buscados na memória. 
Aplicações gráficas, principalmente jogos eletrônicos, são notórias por 
sofrerem deste problema pela elevada quantidade de dados necessários para o
seu funcionamento. Este trabalho propõe a implementação de um motor de jogos 
(\textit{engine}) utilizando um conjunto de práticas de programação, a
 modelagem orientada a dados (MOD), com a finalidade de amenizar este problema. 
Para testar a eficiência da MOD, foram desenvolvidas duas aplicações diferentes, uma 
utilizando a MOD, e outra com programação orientada a objetos, considerada padrão no 
mercado, e seus desempenho foram comparados. Os resultados da comparação foram 
discutidos e uma conclusão foi feita para o trabalho.
